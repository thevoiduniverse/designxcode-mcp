Three Approaches: Design System Context Layer

How should the MCP server deliver persistent design system context to the AI?

A

Single Mega-Resource

One designsystem://context/{fileKey} resource that returns everything — tokens, components, patterns, rules — in a single structured document. The AI reads it once and has the full picture.

Pros

  • Simple to implement — one resource, one fetch
  • AI gets complete context in one read
  • Easy to reason about — no coordination needed

Cons

  • Could be very large (10k+ tokens for big design systems)
  • All-or-nothing — can't fetch just colors if that's all you need
  • Every read hits Figma API (slow, rate limits)
B

Layered Resources + Cache

Multiple resources at different granularity — designsystem://tokens/{fileKey}, designsystem://components/{fileKey}, designsystem://patterns/{fileKey}, designsystem://rules/{fileKey}. An in-memory cache avoids repeated Figma API calls. A build_design_context tool pre-warms the cache and returns a compact summary.

Pros

  • AI can read only what it needs per task
  • Cache makes repeated reads instant
  • Smart detection tool can decide which resources to read
  • Each resource stays within reasonable context size

Cons

  • More complex — multiple resources to coordinate
  • Cache invalidation needs thought (TTL? manual refresh?)
  • AI must know which resources to read and when
C

Smart Tool + Prompt Combo

A get_design_context tool that the AI calls before generating UI code. It fetches tokens + components + patterns, applies intelligent compression (only relevant tokens for the task), and returns a tailored context block. Paired with an MCP prompt template use-design-system that the user can invoke.

Pros

  • Context is task-aware — returns only relevant tokens/components
  • Prompt template gives users a one-command trigger
  • Tool can cache internally + compress intelligently
  • Works naturally with "smart detection" — AI decides when to call

Cons

  • Relies on the AI deciding to call the tool (may forget)
  • More logic in the compression/relevance filtering
  • Prompt templates have limited client support currently
