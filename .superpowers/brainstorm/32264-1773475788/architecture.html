Architecture: Design System Context Layer

How the pieces fit together

System Architecture
┌─────────────────────────────────────────────────────────────────┐
│                        MCP CLIENT (Claude Code / Cursor)        │
│                                                                 │
│  Developer: "Build me a settings page"                          │
│       │                                                         │
│       ▼                                                         │
│  AI detects UI task ──► Calls get_design_context tool           │
│       │                  (or user triggers "use-design-system") │
│       │                                                         │
│       ▼                                                         │
│  AI generates code with correct tokens + existing components    │
└──────────────────────────────┬──────────────────────────────────┘
                               │ MCP Protocol
                               ▼
┌─────────────────────────────────────────────────────────────────┐
│                   DESIGNXCODE MCP SERVER                        │
│                                                                 │
│  ┌─── RESOURCES (always available, cached) ──────────────────┐  │
│  │                                                           │  │
│  │  designsystem://tokens/{fileKey}                          │  │
│  │    → Colors, spacing, typography as CSS vars              │  │
│  │                                                           │  │
│  │  designsystem://components/{fileKey}                      │  │
│  │    → Component inventory with props & variants            │  │
│  │                                                           │  │
│  │  designsystem://patterns/{fileKey}                        │  │
│  │    → Auto-inferred usage patterns from Figma layouts      │  │
│  │                                                           │  │
│  │  designsystem://rules/{fileKey}                           │  │
│  │    → User-defined design rules & overrides                │  │
│  │                                                           │  │
│  └───────────────────────────┬───────────────────────────────┘  │
│                              │                                  │
│  ┌─── TOOLS ─────────────────┼───────────────────────────────┐  │
│  │                           ▼                               │  │
│  │  get_design_context ─► Reads resources ─► Compresses      │  │
│  │    Inputs:              from cache        for task         │  │
│  │    • figma_file_key                                       │  │
│  │    • task_description (optional)                          │  │
│  │    • sections: tokens|components|patterns|rules|all       │  │
│  │                                                           │  │
│  │  set_design_rules ─► Stores user-authored rules           │  │
│  │    Inputs:                                                │  │
│  │    • figma_file_key                                       │  │
│  │    • rules[] (structured)                                 │  │
│  │                                                           │  │
│  └───────────────────────────────────────────────────────────┘  │
│                                                                 │
│  ┌─── PROMPTS ───────────────────────────────────────────────┐  │
│  │                                                           │  │
│  │  use-design-system                                        │  │
│  │    Args: figma_file_key                                   │  │
│  │    → Pre-built instruction that tells the AI to call      │  │
│  │      get_design_context and follow the design system      │  │
│  │                                                           │  │
│  └───────────────────────────────────────────────────────────┘  │
│                                                                 │
│  ┌─── CACHE LAYER ──────────────────────────────────────────┐  │
│  │                                                           │  │
│  │  In-memory cache per fileKey                              │  │
│  │  TTL: 5 minutes (configurable)                            │  │
│  │  Stores: tokens, components, patterns, rules              │  │
│  │  Invalidation: manual via refresh flag OR TTL expiry      │  │
│  │                                                           │  │
│  └───────────────────────────────────────────────────────────┘  │
│                              │                                  │
└──────────────────────────────┼──────────────────────────────────┘
                               │
                    ┌──────────┴──────────┐
                    ▼                     ▼
             ┌─────────────┐      ┌─────────────┐
             │  Figma API  │      │  Local JSON  │
             │  (tokens,   │      │  (rules      │
             │  components,│      │   overrides)  │
             │  file nodes)│      │              │
             └─────────────┘      └─────────────┘
