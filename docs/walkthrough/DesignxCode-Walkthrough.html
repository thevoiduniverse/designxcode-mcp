What is DesignxCode?

DesignxCode is a tool that connects your Figma designs to your code. It reads your design system from Figma (colors, fonts, shadows, components) and turns it into actual working code — automatically.

Think of it like this:

  • You design a Button in Figma with specific colors, sizes, and states
  • DesignxCode reads that Button, understands all its variants, and writes the React code for you
  • If someone changes the design in Figma, DesignxCode detects the drift and can re-sync

The problem it solves: Designers design in Figma. Developers code in VS Code. These two worlds drift apart over time. Colors change, fonts get updated, new components get added — but the code doesn't know. DesignxCode bridges this gap.

How It Works

Figma (your designs)
    ↓
DesignxCode MCP Server (the engine)
    ↓
Your Code (React components, CSS tokens, theme configs)

The MCP server has 12 tools that each do one job:

ToolWhat it doesWhen to use
get_design_contextLoads your full design systemBefore writing any UI code
extract_tokensPulls all design tokensWhen you need token values
extract_stylesGets typography and effectsWhen checking font scales
generate_coded_componentsWrites React components from FigmaWhen you need a new component
generate_theme_configCreates light/dark theme CSSWhen setting up themes
sync_tokens_to_codeGenerates CSS/SCSS/Tailwind filesWhen syncing tokens
audit_component_parityCompares Figma vs codeWhen checking coverage
audit_system_healthHealth score for your design systemWeekly health checks
detect_unused_tokensFinds unused tokens in codebaseWhen cleaning up
export_assetsExports SVG/PNG from FigmaWhen you need icons
generate_sync_prCreates a GitHub PRWhen pushing changes
set_design_rulesSaves custom rulesWhen adding constraints

Setup (Step by Step)

Step 1: Get your Figma token

  1. Go to figma.com → click your profile → Settings
  2. Scroll to Personal access tokens
  3. Click Generate new token → name it "designxcode"
  4. Copy the token (starts with figd_)

Step 2: Get the code

git clone https://github.com/thevoiduniverse/designxcode-workspace.git
cd designxcode-workspace/designxcode-mcp
npm install
npm run build

Step 3: Add your Figma token

Create a file called .env in the designxcode-mcp folder:

FIGMA_ACCESS_TOKEN=figd_your_token_here

Optional — for GitHub features:

FIGMA_ACCESS_TOKEN=figd_your_token_here
GITHUB_TOKEN=ghp_your_github_token_here

Step 4: Connect to Claude Code

Add to your Claude Code MCP config:

{
  "mcpServers": {
    "designxcode-mcp": {
      "command": "node",
      "args": ["/path/to/designxcode-mcp/dist/index.js"]
    }
  }
}

Step 5: Verify

In Claude Code, type /mcp. You should see designxcode-mcp listed.

Step 6: Get your Figma file key

From your Figma URL:

Common Workflows

"I want to generate a Button component"

Ask Claude: "Generate the Button component from my Figma file as React + Tailwind"

You'll get a complete .tsx file with all variants, style maps, and TypeScript types. All values come directly from Figma.

"I want light and dark theme CSS"

Ask Claude: "Generate theme config from my Figma file"

Works even without Figma Enterprise — uses a technique called mode inference that detects light/dark themes from the file tree automatically.

"I want to check if my code matches Figma"

Ask Claude: "Audit component parity for my Figma file against my GitHub repo"

Shows matched, missing, and extra components with coverage percentage.

Problems That Aren't Solved Yet

Great places to start contributing:

1. Property Removal Detection Medium

When a Figma variant removes a property (Ghost button has no background), the differ doesn't catch it. The old value stays in generated code.

File: src/utils/state-differ.ts — diffStyles function

2. Redundant Padding Easy

Parser outputs both padding: "12px 24px" AND individual padding values. Creates duplicate Tailwind classes.

File: src/utils/node-parser.ts

3. Child Node State Bloat Medium

Child elements get duplicate hover/active/disabled classes from scoped state overrides.

File: src/utils/emitter-react-tailwind.ts

4. Token Naming Without Enterprise Hard

Without Figma Enterprise, token names come from node names (ugly). Needs better heuristics or style cross-referencing.

File: src/utils/mode-inference.ts

5. Multi-Axis Variant Differ Hard

Components with 3+ axes (Size × Type × State) get cross-contaminated diffs.

File: src/utils/state-differ.ts

6. Build the Mac App Project

HTML mockup exists, needs to become a real Tauri + React app wired to the MCP server.

Spec: docs/app-screens-spec.md (70 frames, 9 pages)

7. Add Tests Easy

Zero automated tests. Add Vitest and write unit tests for the core utilities.

How to Build a Portfolio Piece

Key Concepts

MCP: Model Context Protocol — lets AI tools talk to external services. DesignxCode is an MCP server that Claude connects to.

Design tokens: The atomic values — colors (#d51e8c), font sizes (16px), shadows. Building blocks.

Mode inference: Our technique for detecting light/dark themes without Enterprise. Walks the file tree, finds variables resolving to different colors, clusters by luminance. Nobody else does this.

Style maps: Clean lookup tables instead of messy inline classes:

const typeStyles = {
  primary: { base: "bg-[#d51e8c]", hover: "hover:bg-[#b8187a]" },
  ghost: { base: "text-[#6b7280]", hover: "hover:bg-[#f3f6fa]" },
};
